# Design Context — Songwriting Assistant

## Design Context

### Users

Professional and semi-pro songwriters using the tool alone, in a focused creative session. They know what syllables and rhymes are — they don't need hand-holding. They want a tool that stays out of the way and supports the flow of writing, not one that imposes itself.

### Brand Personality

**Warm · Intimate · Analog**

The interface should evoke a studio notebook — slightly tactile, slightly literary, like ink on paper. It is a *writing* tool first, an analysis tool second. The analysis panels are marginalia, not dashboards.

### Aesthetic Direction

- Warm off-white backgrounds with aged-paper undertones — never clinical white
- Accent: warm amber-ochre (like old ink or a manuscript annotation) rather than corporate blue
- IBM Plex Sans used expressively: tight tracking on labels, generous weight on headings
- Panels feel like margin notes, not dashboard cards — lighter visual weight, editorial label style
- No gradients, no glow, no dark-theme-first design
- Anti-references: AI chat interfaces, Spotify/music streaming vibes, productivity SaaS (Linear, Jira)

### Design Principles

1. **The writing surface is sacred.** The editor gets the most space, best contrast, and no visual competition from surrounding UI.
2. **Analysis is marginal, not dominant.** The syllable and rhyme panels support the writer; they should feel like handwritten notes in the margin, not control panels.
3. **Warmth over precision.** Prefer warm color, generous line height, and soft borders. Technical precision belongs in the data, not the chrome.
4. **Copy should feel like a collaborator, not a tooltip.** Every label and empty state should sound like something a thoughtful co-writer would say.
5. **Restraint first, expression second.** MVP polish — do not over-decorate. One intentional detail is worth more than five generic embellishments.
